# Googleマップ診断会＋勉強会 — 申込みLP

セミナー申込み用の単一ページLP。
**Cloudflare Pages（静的ホスティング） × GAS（書込み + 残席返却 + リマインド）× Googleスプレッドシート（設定CMS + 申込み管理）** で運用します。

開催ごとに index.html を編集する必要はありません。**スプレッドシートの「設定」シートを書き換えるだけ**で日付・会場・残席などが自動反映されます。

---

| ファイル | 役割 |

| `index.html` | Cloudflare Pages にアップする単一HTML。画像・ファビコンすべて埋め込み済み |

## Aパート：初回セットアップ（1回だけ）

### ① スプレッドシート作成
1. https://drive.google.com → **新規スプレッドシート**
2. 名前を付けて開く（例：`マップ診断会リスト`）

### ② GAS設置（スプレッドシートバインド）
1. スプレッドシートの **拡張機能 → Apps Script** を開く
2. 既存コードをすべて削除し、本プロジェクトの `Code.gs` をペースト → 💾保存
3. 上部ツールバーから関数 `setupSheets` を選び **実行**（初回はGoogleアカウントの権限承認）
   - 「設定」「申込一覧」シートが自動生成されます
4. 右上「**デプロイ**」→「**新しいデプロイ**」→「**ウェブアプリ**」
5. 設定：
   - 実行するユーザー：**自分**
   - アクセスできるユーザー：**全員**
6. 「**デプロイ**」→ 表示された URL（`https://script.google.com/macros/s/.../exec`）を控える
7. スプレッドシートに戻り **📋 MEO Workshop → ⏰ リマインドトリガー設定** を実行（毎朝9時自動）

### ③ HTMLに反映
`index.html` 冒頭の `EVENT_CONFIG.gasUrl` を②で控えた URL に書き換え：

```js
gasUrl: "https://script.google.com/macros/s/XXXXX/exec",
```

### ④ Cloudflare Pages にデプロイ
1. https://dash.cloudflare.com → **Workers & Pages** → **Create** → **Pages** → **Upload assets**
2. プロジェクト名を入力 → `index.html` を **ドラッグ＆ドロップ**（**1ファイルだけでOK**）
3. **Deploy site** → 数秒で公開URL発行

> 画像・ファビコンはすべて `index.html` に埋め込み済み（base64）なので、他のファイルは不要です。

## Bパート：開催ごとの作業（毎回・スプレッドシートだけ）

1. スプレッドシートの **「設定」シート** を開く
2. 以下のセルを書き換え：
   - **開催日** … セルをクリック → カレンダーから日付を選択（`eventId` は自動生成）
   - **開催時間** / **会場名** / **会場住所** / **地図リンク** / **地図埋込URL** など必要な項目
   - **次回開催日**（任意）… 入れると予告枠に表示、空欄なら「調整中」
3. ブラウザでLPを開く → 設定シートの内容が自動反映される

> **index.html の再アップロードは不要**。Cloudflare Pages 側は触らずに開催が切り替わります。

開催ごとに新しい `eventId`（＝開催日）のタブが自動生成され、申込みはそのタブに記録されます。

## Cパート：申込み管理・運用Tips

### シート構成
- **設定** … 開催情報CMS（ここを書き換えるとLP反映）
- **申込一覧** … 全開催の累積ログ（並べ替え用・index側へは反映なし）
- **{eventId}（例：`2026-06-25`）** … 開催別タブ。A列の✓チェックでLPの残席に反映

### 残席カウント
- 開催別タブの A列 ✓ チェックがTRUEの行数をカウント
- キャンセル時は ✓ を外せば即座に残席が1名増える
- LP残席 = `EVENT_CONFIG.capacity` − ✓カウント

### リマインドメール
- 毎朝9時にGASが自動チェック
- 設定シートの「開催日」の3日前に該当開催タブの✓がついた申込者全員にメール送信
- リマインド送信先・送信元の差出人名は `Code.gs` 冒頭の定数で変更可能

### カスタムメニュー（📋 MEO Workshop）
- ↕ 申込一覧：新しい順 / 名前順に並べ替え
- 🔧 初期セットアップ（初回のみ）
- ⏰ リマインドトリガー設定

### 残席APIの直接確認
```
https://script.google.com/macros/s/XXXXX/exec?action=count&eventId=2026-06-25
```
レスポンス例：`{ "ok": true, "eventId": "2026-06-25", "count": 3 }`

## 🛠 トラブルシューティング

| 症状 | 対処 |
|---|---|
| 残席カードが「概算（取得失敗）」 | `EVENT_CONFIG.gasUrl` の値、GASのアクセス権限「全員」、デプロイ済みかを確認 |
| 申込後に行が追加されない | GASエディタ → 実行数でエラー確認。最新版を「**デプロイの管理 → 編集 → 新しいバージョン**」で再デプロイ |
| 開催日を変えても反映されない | ブラウザを **Ctrl+F5** でハード再読み込み。CloudflareのキャッシュTTLがあれば短くする |
| eventIdタブが文字化け | 「設定」シートの「開催日」はカレンダー入力を使用（手入力で全角等を混ぜない） |
| GASを更新したのに動作が古い | GASエディタで「**デプロイ → デプロイの管理 → 編集 → 新しいバージョン**」を毎回必ず実行（URLは同じまま） |
